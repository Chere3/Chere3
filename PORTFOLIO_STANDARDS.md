# Portfolio Repository Standards

Use this as the quality bar for every active public repository.

## Minimum professionalism baseline
- Clear README with:
  - Problem statement
  - Stack
  - Setup instructions
  - Commands (`dev`, `build`, `lint`, `test` when applicable)
- `ROADMAP.md` with:
  - Quick wins
  - Medium-term work
  - Big bets
  - Strategic rewrites
- Contributor onboarding (`CONTRIBUTING.md` or equivalent section)
- Environment docs (`.env.example` + variable explanations) when needed

## Engineering hygiene
- CI for at least lint + build/typecheck
- Predictable script names in package manager config
- Reasonable project structure with clear module boundaries
- Security basics (dependency updates, no secrets in repo)

## Portfolio cadence (monthly)
1. Remove or archive stale repositories with no strategic value
2. Promote best active projects in profile README
3. Ensure featured links, badges, and setup docs are valid
4. Record key updates in `ROADMAP.md`
